## Component Inventory

### 1. AccountDropdown
Reuses data from existing `GET /api/accounts/list`. Renders as a styled select-like list matching existing `AccountSelector` visual pattern (bg-gray-800, border-gray-700, rounded-lg). Shows username, last_login date, and cookie_valid status indicator.

### 2. NoVncViewer
Wrapper around noVNC client library. Props: `wsUrl: string`, `connected: boolean`, `onConnect: () => void`, `onDisconnect: () => void`. Renders a canvas element inside a dark container (bg-gray-950, border border-gray-800, rounded-lg). Shows connection status dot in top-right corner.

### 3. ConnectionStatusBadge
Small badge showing VNC connection state. Three states:
- **Connected**: green-500 dot + "Conectado" text in green-200
- **Connecting**: yellow-500 dot + "Conectando..." text in yellow-200, with pulse animation
- **Disconnected**: gray-500 dot + "Desconectado" text in gray-400

### 4. CaptureButton
Primary CTA. Full-width button matching existing button patterns (py-3 px-4 rounded-lg font-bold). States:
- **Disabled** (no session): bg-gray-800 text-gray-500 cursor-not-allowed
- **Ready**: bg-blue-600 hover:bg-blue-500 text-white
- **Loading**: bg-blue-600 text-white with Loader2 spin animation
- **Success**: Temporarily shows CheckCircle icon + "Cookies Capturados!" for 3 seconds

### 5. StatusFeedback
Reuses existing pattern from LoginButton.tsx:
- **Success**: bg-green-900/30 border-green-800 text-green-200, CheckCircle icon
- **Error**: bg-red-900/30 border-red-800 text-red-200, AlertTriangle icon
- Shows problem description + actionable next step

## Interaction States

### Page Load
1. Page renders with account selector visible, noVNC area shows empty state
2. User selects an account from the dropdown
3. Browser session starts automatically (or via explicit "Iniciar Sessao" button -- Claude's discretion per D-discretion)
4. noVNC viewer connects and shows the Browserless Chromium with Instagram login page

### Cookie Capture Flow
1. User completes Instagram login manually in the embedded browser (including 2FA)
2. User clicks "Capturar Cookies" button
3. Button enters loading state (spinner)
4. API extracts cookies, validates sessionid exists, encrypts, saves
5. On success: StatusFeedback shows success banner, account marked cookie_valid=true
6. On error: StatusFeedback shows error banner with specific problem and retry guidance

### Error States
| Error | Visual | Recovery |
|-------|--------|----------|
| Browserless unreachable | Red banner: "Nao foi possivel conectar ao Browserless. Verifique se o container esta rodando." | User checks Docker containers |
| No sessionid cookie | Red banner: "Cookie sessionid nao encontrado. Complete o login no Instagram antes de capturar." | User completes login in embedded browser |
| Account save failed | Red banner: "Erro ao salvar cookies. Tente novamente." | Retry button |
| No accounts in DB | Empty state in account selector area | User adds account via existing flow |
