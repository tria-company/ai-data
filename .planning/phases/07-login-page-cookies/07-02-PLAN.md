Create `app/admin/login-session/page.tsx` as a `'use client'` component. This is a single-file page containing all UI logic (AccountDropdown, BrowserViewer, ConnectionStatusBadge, CaptureButton, StatusFeedback are inline sections, not separate component files). Follow the UI-SPEC design contract exactly.

**Page structure (per UI-SPEC):**

```
'use client'
import { useState, useEffect, useRef, useCallback } from 'react';
import { io, type Socket } from 'socket.io-client';
import { Loader2, CheckCircle, AlertTriangle, ArrowLeft } from 'lucide-react';
```

**State management:**
- `accounts`: Array fetched from `/api/accounts/list`
- `selectedAccountId`: string | null
- `connectionStatus`: 'disconnected' | 'connecting' | 'connected'
- `captureStatus`: 'idle' | 'loading' | 'success' | 'error'
- `errorMessage`: string
- `socketRef`: useRef<Socket | null>
- `imgRef`: useRef<HTMLImageElement>
- `containerRef`: useRef<HTMLDivElement> (for coordinate scaling)
- `viewportSize`: { width: number, height: number } (received from server on connect)

**On mount:**
1. Fetch `GET /api/socket` to initialize Socket.io server
2. Fetch `GET /api/accounts/list` to populate dropdown
3. Create socket.io client connection: `io({ path: '/api/socket/io' })`
4. Register socket event listeners:
   - `'screencast-frame'`: set imgRef.current.src = `data:image/jpeg;base64,${data}`
   - `'session-status'`: update connectionStatus, store viewportWidth/viewportHeight
   - `'session-error'`: set connectionStatus='disconnected', show error
   - `'cookies-captured'`: POST cookies to `/api/accounts/import-cookies` with selectedAccountId, then show success/error
   - `'cookies-error'`: set captureStatus='error', show message
5. Return cleanup: disconnect socket, emit 'stop-session'

**Account selection:**
When user selects an account from dropdown:
1. Set selectedAccountId
2. If socket connected, emit `'stop-session'` (cleanup previous)
3. Set connectionStatus='connecting'
4. Emit `'start-session'`

**Browser viewer interaction (per D-01, D-02, Pitfall 1):**
The viewer is an `<img>` tag inside a container div. Coordinate scaling is CRITICAL:
- `scaleX = viewportSize.width / containerRect.width`
- `scaleY = viewportSize.height / containerRect.height`

Event handlers on the container div:
- `onClick`: emit `'mouse-click'` with scaled coordinates
- `onMouseMove`: emit `'mouse-move'` with scaled coordinates (throttle to ~30ms)
- `onWheel`: emit `'scroll'` with deltaX, deltaY
- `onKeyDown` (on a wrapper div with tabIndex=0):
  - For printable single characters: emit `'key-type'` with `{ text: e.key }`
  - For special keys (Enter, Tab, Backspace, ArrowUp/Down/Left/Right, Escape): emit `'key-press'` with `{ key: e.key }`
  - Call `e.preventDefault()` to prevent browser default actions

**Cookie capture (per D-05, D-06, D-07):**
When "Capturar Cookies" button clicked:
1. Set captureStatus='loading'
2. Emit `'capture-cookies'`
3. On `'cookies-captured'` event: POST to `/api/accounts/import-cookies` with `{ accountId: selectedAccountId, cookies }`
4. If POST returns 200: set captureStatus='success'
5. If POST returns error: set captureStatus='error', set errorMessage from response
