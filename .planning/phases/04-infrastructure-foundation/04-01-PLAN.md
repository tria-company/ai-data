Create a multi-stage Dockerfile for the Next.js app. Per D-02, Bull Board runs inside the app, so no separate container for it.

**Stage 1 — deps:**
```dockerfile
FROM node:20-alpine AS deps
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --ignore-scripts
```

**Stage 2 — builder:**
```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
ENV NEXT_TELEMETRY_DISABLED=1
RUN npm run build
```

**Stage 3 — runner:**
```dockerfile
FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production
ENV NEXT_TELEMETRY_DISABLED=1

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs
EXPOSE 3000
ENV PORT=3000
ENV HOSTNAME="0.0.0.0"

HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:3000/ || exit 1

CMD ["node", "server.js"]
```

IMPORTANT: The runner stage uses Next.js standalone output. This requires adding `output: 'standalone'` to next.config.ts (done in Task 2).

Note: @sparticuz/chromium is NOT needed in the Docker container since we use Browserless via WebSocket. The Dockerfile intentionally does NOT include chromium binaries.

**1. Create docker-compose.yml** at project root. Per D-01, single .env shared via env_file. Per D-03, this is the production compose file. Per D-04, Redis is ephemeral (no volumes). Per D-06, Browserless container included.

```yaml
services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    env_file:
      - .env
    depends_on:
      redis:
        condition: service_healthy
      browserless:
        condition: service_started
    networks:
      - scraper-net
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 3
    networks:
      - scraper-net
    restart: unless-stopped

  browserless:
    image: ghcr.io/browserless/chromium:latest
    ports:
      - "3333:3000"
    environment:
      - TIMEOUT=120000
      - CONCURRENT=2
      - HEALTH=true
    networks:
      - scraper-net
    restart: unless-stopped

networks:
  scraper-net:
    driver: bridge
```

Key points:
- Redis has NO volumes (per D-04, ephemeral)
- Redis healthcheck ensures app waits for Redis to be ready
- Browserless exposes on port 3333 (its internal port is 3000, mapped to 3333 to avoid conflict with app)
- Per D-02, Bull Board is NOT a separate container — it's mounted inside the app (Plan 02)
- Network `scraper-net` ensures all containers can reach each other by service name
