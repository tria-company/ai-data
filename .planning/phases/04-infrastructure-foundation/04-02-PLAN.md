const connection = {

**1. Create app/api/admin/queues/route.ts** — Next.js catch-all route handler for Bull Board.

Per D-02, Bull Board is mounted inside the Next.js app using @bull-board/next-adapter.

```typescript
import { serverAdapter } from '@/lib/bullboard';

export const dynamic = 'force-dynamic';

const handler = serverAdapter.getHandler();

export { handler as GET, handler as POST };
```

Note: The `@bull-board/next-adapter` NextAdapter handles all routing internally. The route.ts just delegates to it.

IMPORTANT: This route file needs to be a catch-all to handle Bull Board's sub-routes. If Bull Board assets don't load, the file may need to be moved to `app/api/admin/queues/[[...slug]]/route.ts`. Start with the simple path and adjust if needed.

**3. Create docker-compose.override.yml** — Per D-03, dev overrides for hot-reload and debug.

```yaml
# Dev overrides - automatically merged with docker-compose.yml
# Usage: docker compose up (automatically picks up both files)
services:
  app:
    build:
      target: builder
    command: npm run dev
    volumes:
      - .:/app
      - /app/node_modules
      - /app/.next
    environment:
      - NODE_ENV=development
    ports:
      - "3000:3000"
      - "9229:9229"

  redis:
    ports:
      - "6379:6379"
```

Key points:
- Overrides `app` to use builder stage (has source code + node_modules) and runs `npm run dev` instead of standalone
- Volume mounts `.:/app` for hot-reload, with anonymous volumes for node_modules and .next to avoid conflicts
- Exposes port 9229 for Node.js debugger
- Redis port exposed for local tooling (redis-cli, RedisInsight)
