Job data interface:

```typescript
  projetoId: string | null;  // Project ID for data isolation

}
```

Worker processor logic — decomposed from `scrapeAccounts()` in scraper.ts:

1. **Account selection loop (per D-05):** Try accounts sequentially within the SAME job:
   ```
   let account = await selectAccount();
   while (account) {
     try { ... scrape logic ... return result; }
     catch (error) {
       if (isCookieError(error)) {
         await markAccountInvalid(account.id);
         account = await selectAccount(); // try next account
         continue;
       }
       throw error; // non-cookie errors trigger BullMQ retry
     }
   }
   ```

2. **No accounts available (per D-06, ACCT-04):** When `selectAccount()` returns null after the loop:
   ```
   await profileScrapeQueue.add('profile-scrape', job.data, {
     delay: 30 * 60 * 1000, // 30 minutes
   });
   console.log(`[profile-worker] No accounts available, re-queued with 30min delay`);
   return { status: 'requeued', reason: 'no_accounts_available' };
   ```

3. **Browser + cookie setup** (adapted from scraper.ts lines 53-97):
   - `const browser = await getBrowser();`
   - `const page = await browser.newPage();`
   - Decrypt cookies from `account.session_cookies` using the same logic as scraper.ts lines 54-66
   - Sanitize and set cookies using the same pattern as scraper.ts lines 73-95
   - CRITICAL: Wrap all browser work in `try { ... } finally { await browser.close(); }` per Pitfall 5

4. **Profile scraping** (adapted from scraper.ts lines 101-167):
   - Navigate to `https://www.instagram.com/${username}`
   - Add random delay: `await delay(Math.random() * 2000 + 1000)`
   - Check if private: `await checkIfPrivate(page)`
   - Extract bio: `await extractBio(page)` — upsert to `profile_bio` table with `projetoId`
   - Extract highlights: `await extractHighlights(page, username)` — upsert each to `profile_highlights` table
   - Scroll and extract posts: `await scrollToBottom(page, username, maxPosts)` then `await extractPostsData(page, username, maxPosts)`
   - Save posts to `scrappers_contents` table using same upsert pattern as scraper.ts lines 253-289
   - Report progress via `job.updateProgress()` at key milestones (bio done: 20%, highlights: 40%, posts: 80%, enqueue: 100%)

5. **Enqueue post-details jobs (QUEUE-03):**
   After extracting post list, bulk-enqueue individual post jobs:
   ```typescript
   const postJobs = posts.map(post => ({
     name: 'post-details',
     data: {
       postId: post.postId,
       postUrl: post.postUrl,
       mediaType: post.mediaType,
       isCarousel: post.isCarousel,
       username,
       projetoId,
     },
     opts: {
       attempts: 3,
       backoff: { type: 'exponential' as const, delay: 30000 },
     },
   }));
   await postDetailsQueue.addBulk(postJobs);
   console.log(`[profile-worker] Enqueued ${postJobs.length} post-details jobs for @${username}`);
   ```

6. **Worker options (per D-08, D-09):**
   ```typescript
   const worker = new Worker('profile-scrape', processor, {
     connection,
     concurrency: 1,  // One job at a time (per Pitfall 4 — Browserless CONCURRENT=2)
     limiter: { max: 2, duration: 60000 }, // Safety net: max 2 jobs/min (per D-08)
   });
   ```

7. **Error handler + graceful shutdown:**
   ```typescript
   worker.on('error', (err) => console.error('[profile-worker] Error:', err));
   worker.on('completed', (job) => console.log(`[profile-worker] Job ${job.id} completed`));
   worker.on('failed', (job, err) => console.error(`[profile-worker] Job ${job?.id} failed:`, err.message));

   async function shutdown() {
     console.log('[profile-worker] Shutting down...');
     await worker.close();
     process.exit(0);
   }
   process.on('SIGTERM', shutdown);
   process.on('SIGINT', shutdown);

   console.log('[profile-worker] Started, waiting for jobs...');
   ```
