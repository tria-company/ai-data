<task type="auto">
  <name>Task 2: Create extraction functions for likes and comments</name>
  <files>lib/extraction.ts</files>
  <action>
Add two new exported async functions to lib/extraction.ts, following the exact same pattern as extractVideoUrl and extractCarouselImages (open new page tab, navigate to post URL, extract data, close tab, return data):

1. **extractPostLikes(page: Page, postUrl: string): Promise&lt;string[]&gt;**
   - Opens a new browser tab, navigates to the post URL
   - Waits for page load (delay 2000ms like other functions)
   - Clicks the likes count element to open the "liked by" modal. The likes count on Instagram is typically a button/link near the post showing "X likes" or "Liked by ... and X others". Look for selectors like:
     - `button` or `a` containing text matching /liked by|curtido por|(\d+) likes?|(\d+) curtidas?/i
     - `section` elements near the post that contain like counts
   - Once the modal opens, scroll within the modal to load more likers (Instagram lazy-loads the list). Scroll up to 5 times with 1s delays.
   - Extract all usernames from the modal. Likers are typically shown as links to profiles: `a[href^="/"]` within the modal dialog, where the text content is the username.
   - Close the tab and return array of username strings.
   - On any error (modal doesn't open, no likes visible, timeout), return empty array []. Do NOT throw.

2. **extractPostComments(page: Page, postUrl: string): Promise&lt;Array&lt;{username: string, text: string}&gt;&gt;**
   - Opens a new browser tab, navigates to the post URL
   - Waits for page load (delay 2000ms)
   - Comments are typically visible on the post page directly. Look for comment elements:
     - Each comment is usually in a `ul` list within the comments section
     - Each comment item contains a username link (`a[href^="/"]`) and the comment text in a `span` sibling
     - The first "comment" is often the post caption by the author -- skip it (check if username matches the post author from the URL)
   - If there's a "Load more comments" / "Ver mais comentarios" button/link, click it up to 3 times with 1.5s delays to load more.
   - Extract username and comment text pairs. Use page.evaluate to scan the DOM:
     - Find all comment containers (li elements or div with role="button" containing user links)
     - For each, get the first `a` tag href as username (strip leading /) and the adjacent span text as comment
   - Close the tab and return array of {username, text} objects.
   - On any error, return empty array []. Do NOT throw.

IMPORTANT: Instagram's DOM changes frequently. Make the selectors resilient by trying multiple strategies:
- Try specific selectors first, fall back to broader text-content matching
- Use aria-labels where possible (they're more stable)
- Log a warning to console if extraction yields 0 results (helps debug selector changes)
