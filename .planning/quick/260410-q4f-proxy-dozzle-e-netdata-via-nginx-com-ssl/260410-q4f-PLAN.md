Editar `nginx/default.conf` para adicionar 3 novos blocos location DENTRO do server block HTTPS (listen 443 ssl), ANTES do `location /` existente. NÃO remover, modificar ou reordenar nada que já existe (HSTS, ssl_certificate, location /, server HTTP:80 com acme-challenge).

Adicionar no topo do arquivo (fora dos servers), um bloco `map` para gerenciar o header Connection conforme o valor de Upgrade (padrão recomendado para WebSocket):

```nginx
map $http_upgrade $connection_upgrade {
    default upgrade;
    ''      close;
}
```

Dentro do server HTTPS, ANTES do `location /` atual, adicionar:

```nginx
    # Dozzle — live log viewer sob /logs/
    # Dozzle usa DOZZLE_BASE=/logs (ver docker-compose.yml) para servir assets corretamente
    location /logs/ {
        proxy_pass http://dozzle:8080/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 3600s;
        proxy_buffering off;
    }

    # Netdata — monitoring dashboard sob /netdata/
    # Padrão oficial: https://learn.netdata.cloud/docs/netdata-agent/configuration/running-the-netdata-agent-behind-a-reverse-proxy/nginx
    location = /netdata {
        return 301 /netdata/;
    }
    location ~ /netdata/(?<ndpath>.*) {
        proxy_pass http://netdata:19999/$ndpath$is_args$args;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Server $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_http_version 1.1;
        proxy_pass_request_headers on;
        proxy_set_header Connection "keep-alive";
        proxy_store off;
    }
```

Notas de implementação:
- O `map $http_upgrade $connection_upgrade` PRECISA ficar no escopo `http` — em `conf.d/default.conf` isso significa no topo do arquivo, fora de qualquer `server { }`. Nginx inclui esse arquivo dentro do http block do nginx.conf principal.
- Dozzle roda internamente em 8080 (confirmado em docker-compose.yml `9999:8080`). Nome DNS no docker network = `dozzle`.
- Netdata roda internamente em 19999. Nome DNS = `netdata`.
- A trailing slash em `proxy_pass http://dozzle:8080/;` é crítica: faz o nginx strip do prefixo `/logs/` antes de passar. Com `DOZZLE_BASE=/logs` no Dozzle, ele espera receber os paths como `/logs/...` — NÃO, espera receber os paths normais porque o `DOZZLE_BASE` é só para gerar os links/assets corretamente. A combinação trailing-slash-proxy-pass + DOZZLE_BASE é o padrão recomendado na documentação do Dozzle para subpath behind reverse proxy.
- Para Netdata, o padrão usa regex named capture `(?<ndpath>.*)` e reconstrói a URL com `$is_args$args` para preservar query strings. NÃO usar trailing slash no proxy_pass quando há regex capture.
- `proxy_buffering off` no Dozzle é importante para que os logs aparecem em tempo real (SSE/WS streaming).
- NÃO adicionar auth basic ou qualquer outra restrição — mantém o comportamento atual (público, mesmo risk level do acesso via porta direta).
